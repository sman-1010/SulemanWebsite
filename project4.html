AI THAT PLAYS CLASSIC SNAKE GAME

Some more about the project

Why this project?

This project was done as the semester project in the undergraduate course ME461-Mechatronics.

Abstract

An AI was made that works on the A* search algorithm, this AI was deployed to play the classic Snake Game on PyGame and also playing it on hardware which was a LED display setup with Arduino, the AI made in python communicated with the arduino through PySerial Library. The AI was further improved on my own time to be smarter by having an artificial weighted grid to improve performance,

Languages and Libraries Used

  • Python
  • PySerial
  • PyGame
  • Arduino

Project Details



Classic Snake Game Implementation

What does each part do?

  • Arduino Uno: The chosen microcontroller where everything was programmed
  • 8x8 LED Grid: Acted as display for game
  • Blue Potentiometer: Controlled speed of the game
  • Black Button: Deploys random obstacles into game
  • Joystick: For the user to control the snake

The classic snake game is implemented now, what's next?

  • An AI is made in Python that communicates with the hardware with serial communication using PySerial.


Implementing vanilla A* to play on Hardware


  • The AI could play much faster than a human could

  • Room for Improvement:
    • True potential of AI could not be seen as it limited to 8x8 grid

    • The AI did not plan ahead, it only aimed for the food, regardless if it would die after getting it

    • The AI was unaware of it's own dynamics that it's body is not a static obstacle but a dynamic one over time.

Improvement - A Dynamic Weighted Grid instead of Unit Cost Grid

  • Nodes neighboring the Edges had a high cost
    • These nodes were to be avoided to prevent the snake trapping itself, leaving an exit route for difficult situations

  • Nodes neighboring Snake Body had no cost
    • Encouraged to visit these nodes for more compact & ordered movement situations
    • Decreases gaps between Snake Body to reduce chance of food spawning encircled by snake body

  • All other non-obstacle nodes have a medium cost
    • Prioritized more than Edge Nodes , but less than Nodes neighboring Snake Body


Improvement - Think Ahead, Stalling may be a better option

  • Although A* provides a path to food, it isn’t always the best decision to pursue it immediately

  • If the food is neighbored by at least 3 non-available nodes (obstacles & snake body), it would be a safer choice to stall and move further away from node in an ordered manner until there is a certain exit route after eating the food.

  • It moves away based on shared influence from heuristic from food being maximum and cost of neighboring node being minimum.

Result of the Improvement

  • DANCE MIN
    • Un-weighted Grid and move closer to food when there is no A* path

  • DANCE MAX
    • Un-weighted Grid and move further away from food when there is no A* path

  • WEIGHTED GRID
    • Weighted Grid and move based on minimum cost of neighboring node and maximum distance from food when there is no A* path


Demonstration of Improved Snake AI

The AI now plays more similar to a human:

  • It moves close to its body to take the least amount of volume on the grid

  • It stalls in a very compact form to gain more time to plan a future escape

  • It is aware of it's own tail clearing the way in few coming steps
